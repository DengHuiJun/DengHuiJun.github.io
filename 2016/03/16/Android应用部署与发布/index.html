Android应用部署与发布

主要流程

  1. 用户协议(本地，网络，API，开源库)
  2. 签名文件
  3. 程序图标
  4. 其他（版本号确定，截图，宣传视频，文案，合作首发）

需要配置的项目（现在一般自动化设置）

  1. 清理日志调用，版本代号（versionCode）与版本名称（versionName）
  2. 编译程序，签名程序
  3. 发布前彻底测试一遍
  4. 检查资源是否最新 （Asset）
  5. 确保远程服务器和服务准备就绪
  6. 其他检查项（第三方API的Key，用户协议知识产权，公司，个人）

差异化功能的检查

  1. 不同渠道的功能
  2. 不同Android版本的业务功能
  3. 不同手机型号的业务功能（如：魅族）

代码混淆

  1. 代码混淆是否开启
  2. 代码混淆文件配置是否正确（第三方库有相应的配置文件）

ProGuard（一种工具）混淆

  1. 去掉语义
  2. 清理无用代码
  3. 压缩与优化

问题1：代码调试难
问题2: 反射受到阻碍

gradle中使用proguard

找到build.gradle

默认显示为

buildTypes {
    release {
        minifyEnabled false
        proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
    }
}

把 false 改为true

特定的第三方库和API有特定的配置，否则打包失败

Retrace 来调试

参考：http://www.jianshu.com/p/0202845db617

反编译

  • .apk 改后缀为 .zip
  • 解压 -> dex
  • 通过 dex2jar.bat 将 dex -> jar
  • JD-GUI 来查看类似Java代码

加密的选择

  1. 是否需要加密
  2. 加密是否支持ART模式
  3. 加密后需要测试各个版本的支持情况
  4. x60渠道特殊情况（不同渠道的加密方式）

发布前的全面回归测试

  1. 屏幕分辨率
  2. 不同版本
  3. 核心功能（主流程）
  4. 向前兼容性
  5. 新版本检测添加的功能
  6. 用户统计
  7. 推送
  8. 分享
  9. 广告系统
  10. 名称，网址，logo
  11. 其他
