重识Java(8)

记

花了2天时间看了下 Java：A Beginner‘s Guide，Sixth Editon 的中文版 基于 Java 8.整理出了一些还不太熟悉的知识点，记录如下。

Chapter One:Java基础

  • 字节码：源代码被Java编译器编译后，生成字节码（bytecode），字节码是在JVM上运行时的指令。
  • 面向对象三大特性：封装，多态，继承
  • JVM要在任何对象被创建之前调用main（）
  • 使用代码块不会降低运行时效率（反而可能提高效率）
  • Java关键字保留了const和goto，但不能使用

Chapter Two:数据类型与运算符

  • 强类型，解释型
  • 4种整数类型：byte 8，short 16，int 32，long 64 （注意区分char，保存整数的二进制）
  • 2种浮点型：float 32，double 64
  • char是无符号16位类型（0～65536）
  • 字符使用Unicode
  • 字面值注意加符号（如：12L，11.3F，0xFF，014，0b10等）
  • 变量在进入作用域时被创建，在离开作用域时被销毁

Chapter Three:程序控制语句

  • JDK 7之前switch表达式必须是byte，short，int，char类型或枚举，JDK 7开始，加入了String
  • 不允许重复的case
  • 通过标记代码块和break 标记，可达到goto效果
  • continue可以指定标记来描述要继续执行的包含它的循环（从内部跳转至外部循环）

Chapter Four:类，对象和方法

  • finalize（）在对象最后被垃圾回收器销毁之前被调用，确保对象完全终止（该方法定义在Object中）

Chapter Five:其他数据类型与运算符

  • for-each形式的for循环是最佳选择（只读情况下）
  • String类型常用方法：equals，length，chatAt，compareTo，indexOf，lastIndexOf
  • 字符串的拼接使用StringBuffer，String一旦创建是不可变的
  • 位移运算符可用作乘除2，左移1个乘2，右移1个除2

Chapter Six:方法和类详解

  • 默认访问设置本质上是公有的
  • Java中，签名（signature）指的是方法名及其形参列表
  • 可变长度实参，（数据类型 … 变量名），使用重载，注意避免产生歧义

Chapter Seven:继承

  • super 指向超类的引用，this 当前类的引用
  • Java中运行时多态，体现在重写和重载

Chapter Eight:包和接口

  • Java中，protected创建的成员可以由包中的任何代码访问，但是在包外只能由子类访问
  • 接口中可定义共享常量（有争议）
  • 接口能被接口继承（extends）
  • JDK8开始，接口可以定义默认方法（使用default关键字）
  • JDK8开始，接口可定义static方法，static接口方法不能被实现类继承，也不能被子接口继承

Chapter Nine:异常处理

  • 异常的产生：1.某些内部错误发生，JVM可以产生异常；2.有程序代码中的错误导致的标准异常；3.使用throw语句手动生成异常
  • 一般程序不捕获异常，它会被JVM捕获，JVM默认异常处理会终止程序执行，并显示堆栈跟踪和出错消息
  • JDK 7 开始 增加try－with－resources自动资源管理；增加多重捕获

Chapter Ten:使用I/O

  • 使用字节流读写文件（FileInputStream，FileOutputStream）
  • 读取二进制数据使用DataInputStream和DataOutputStream
  • Java还提供随机访问文件RandomAccessFile（）
  • 字符流BufferedReader和PrintWriter
  • 可通过Scanner读取控制台输入数据

Chapter Eleven:多线程程序设计

  • isAlive()
  • Fork/Join框架支持所谓的“并行编程”

Chapter Twelve:枚举，自动装箱，静态导入和注释

  • enum关键字，默认为公有的静态成员
  • 枚举是类类型，预定义values（）返回包含枚举常量的数组和valueOf（String str）返回一个枚举常量
  • 每一个枚举常量都是一个其枚举类型的对象
  • 枚举不能继承另一个类；enum不能是超类
  • ordinal（）返回该枚举常量在列表中的位置的值，compareTo（）比较相同枚举中的两个常量的顺序值。

枚举示例：

  • 8大类型封装器：Double，Float，Long，Integer，Short，Byte，Character，Boolean
  • 从类型封装器中提取值的过程称为拆箱
  • 把值封装在对象中的过程称为装箱
  • 使用静态导入时，可直接通过静态成员的名称来引用它们（如：import static java.lang.Math.sqrt 之后可直接使用sqrt（））
  • Java提供了一项可以在源文件中嵌入附加信息的功能，这些信息称为注解。
  • 不带形参的注解称为标记注解（如：@Override ）
  • JDK8 开始可以对类型用法进行注解，如强制转换类型或者方法返回类型

注解创建示例：

Chapter Thirteen:泛型

  • 核心：参数化类型
  • 优点：自动处理传递给其类型形参的数据类型
  • 泛型只能用于引用类型
  • 通配符 “？”
  • 泛型接口
  • JDK5中引入的泛型
  • JDK7开始，可使用“<>” 菱形运算符进行类型推断
  • 擦除特性
  • 歧义错误

Chapter Fourteen:lambda表达式和方法引用

  • 称为闭包
  • 函数式接口

简单示例：

  • 提供了一种将代码片段转换为对象的方法
  • 抛出受检异常需要在接口中写明
  • 方法引用，使用”::”运算符
  • 构造方法引用，使用 MyClass::new

Java并发编程之初篇

不同进程之间的通信机制

  1. 套接字
  2. 信号处理器
  3. 共享内存
  4. 信号量
  5. 文件

为什么引入并发

  • 资源利用率
  • 公平性（同等的使用权）
  • 便利性（多任务）

性能问题

  • 服务时间长
  • 响应不灵敏
  • 吞吐率过低
  • 资源消耗过高
  • 可伸缩性较低
  • 上下文切换频繁

线程安全性

同一个进程中的所有线程都将共享进程的内存地址空间，因此这些线程都能访问相同的变量并在同一个堆上分配对象，此时没有相关同步机制，会导致不可预测的结果。


一个对象是否需要线程安全的，取决于它是否同时被多个线程访问

同步

  • synchronized 独占的加锁方式
  • volatile类型的变量
  • 显示锁（Lock）
  • 原子变量

实际并发编程中，尽可能地使用现有的线程安全对象来管理类的状态，如java.util.concurrent包中的类

为了确保某个线程写入该变量的值对于其他线程来说都是可见的，要求在访问某个共享且可变的变量时所有线程在同一个锁上同步

相关概念

  • 重排序
  • 重入
  • 发布（Publish）与逸出（Escape）
  • 线程封闭

Android应用部署与发布

主要流程

  1. 用户协议(本地，网络，API，开源库)
  2. 签名文件
  3. 程序图标
  4. 其他（版本号确定，截图，宣传视频，文案，合作首发）

需要配置的项目（现在一般自动化设置）

  1. 清理日志调用，版本代号（versionCode）与版本名称（versionName）
  2. 编译程序，签名程序
  3. 发布前彻底测试一遍
  4. 检查资源是否最新 （Asset）
  5. 确保远程服务器和服务准备就绪
  6. 其他检查项（第三方API的Key，用户协议知识产权，公司，个人）

差异化功能的检查

  1. 不同渠道的功能
  2. 不同Android版本的业务功能
  3. 不同手机型号的业务功能（如：魅族）

代码混淆

  1. 代码混淆是否开启
  2. 代码混淆文件配置是否正确（第三方库有相应的配置文件）

ProGuard（一种工具）混淆

  1. 去掉语义
  2. 清理无用代码
  3. 压缩与优化

问题1：代码调试难
问题2: 反射受到阻碍

Android屏幕适配

支持屏幕尺寸

屏幕尺寸

  • 指屏幕的对角线的长度，单位是英寸，1英寸 ＝ 2.54 cm

屏幕分辨率

  • 指在横纵向上的像素点数，单位是px，1px ＝ 1个像素点
  • 一般以纵向像素＊横向像素，如1920*1080

屏幕像素密度

  • 指每英寸上的像素点数，单位dpi，即“dot per inch”的缩写
  • 屏幕像素密度与屏幕尺寸和屏幕分辨率有关
  • dpi ＝ （纵向像素的平方＋ 横向像素的平方）开根号 ／ 屏幕尺寸
  • px：构成图像的最小单位
  • 160dp 下 ：1dp ＝ 1px
  • sp：Scale－Independent Pixels，可以根据文字大小首选项进行放缩, 推荐，12，14，18，22 设置大小

mdpi，hdpi，xdpi，xxdpi

  • weight 最后的宽度 ＝ 原来的宽度 ＋ 剩余空间所占百分比的宽度
  • 尽量使用相对布局
  • Android3.2之前使用large限定符，现在使用sw600dp，最小宽度
不同尺寸设备应用不同的布局,一般采用2套,来适配不同屏幕尺寸设备：

如：1 res/layout/main.xml

2 res/layout-sw600dp/main.xml
可使用布局别名来引用

  • 使用屏幕方向限定符
  • res/values-sw600dp-land/layouts.xml 水平方向
  • res/values-sw600dp-port/layouts.xml
  • 使用.9.png,自动拉伸，AndroidStudio已经集成生成器

支持屏幕密度

px 与 dp 的计算
  • 一种思路为：以某种分辨率为标准，如 480X320 ，分成相应的份数，生成所有已知不同分辨率设备下，values-纵像素 x 横像素，对应的文件 ，通过使用资源文件来引用不同分辨率不同的dp值。

  • 缺陷：必须提供所有分辨率设备资源文件

  • Android Studio中可通过预览所有设备，查看布局的适配性

提供备用位图
  • 注意切图来匹配不同分辨率设备，减少内存占用
  • drawable放背景，其他等图片资源
  • mipmap一般放图标资源

实施自适应用户界面流程

  • 针对平板与手机，例子，新闻类应用，结合Fragment
  • 高清设计图，先高分辨率，再切低分辨率图片

注：以上主要学习自慕课网视频。
